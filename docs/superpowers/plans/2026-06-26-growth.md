# План: рост — сервис и привлечение клиентов

> Расширение PWA «Записи клиентов». Нежно-розовый стиль, офлайн, без сервера.
> Процесс: TDD для логики, ревью каждого шага, передеплой в конце.

Пользователь выбрал ВСЕ предложенные функции.

## Навигация (остаётся 6 вкладок)
Вкладку «Рассылка» расширяем в раздел **«Продвижение»** (рассылка + пост «Свободные
окошки» + акции + прайс-лист). Сервисные функции — на Главной и в карточках.

## Модель данных (аддитивно)
- client: `birthday` (''), `tags` (string[] []), `referredBy` ('' — имя/кто привёл).
- meta: `loyaltyEvery` (5), `reviewLink` (''), `businessName` ('').

## Задачи

### G1: Данные и логика (TDD)
**Files:** `src/db/clients.js`(+поля), `src/lib/loyalty.js`(new), `src/lib/posts.js`(new), `src/lib/messages.js`(new, шаблоны), `src/lib/stats.js`(+sourceStats, upcomingBirthdays), тесты.
- clients: addClient/updateClient принимают/сохраняют `birthday`, `tags`, `referredBy` (дефолты '' / [] / '').
- `loyalty.js`: `loyaltyInfo(visitCount, every=5)` → `{ count, isRegular, visitsToDiscount, discountNow }` (discountNow когда (count % every === 0 && count>0)).
- `posts.js`: `buildFreeSlotsPost(freeSlots, dateLabel, businessName)` → текст для сторис: «{businessName}\nСвободно {dateLabel}: 14:00, 16:00\nЗапись в директ 💌». Граничные: нет окон → «всё занято».
- `messages.js`: `reviewRequestText(appt, reviewLink)`; `birthdayText(client)`; `promoTemplates` (массив готовых акций-шаблонов). Чистые функции.
- `stats.js`: `sourceStats(clients)` → `[{source,count}]` (пустой источник → «Не указан»), отсортировано по убыванию; `upcomingBirthdays(clients, days=30)` → клиенты с ДР в ближайшие N дней (учёт месяца/дня, без года).
- Тесты на всё.

### G2: Настройки + расширение клиента
**Files:** `src/components/SettingsView.jsx`, `src/components/ClientForm.jsx`, `src/components/ClientsView.jsx`, `src/lib`(loyalty), `src/styles.css`.
- Настройки: секция «Лояльность и продвижение» — `loyaltyEvery` (number), `reviewLink` (url), `businessName` (text) → setMeta.
- ClientForm: поля «Дата рождения» (input date), «Теги» (ввод через запятую → массив, показ чипами), «Кто привёл» (text). Всегда передавать ВСЕ поля в updateClient.
- ClientsView: в раскрытии клиента показать: бейдж лояльности (loyaltyInfo по числу визитов из истории), теги-чипы, ДР; кнопка «🔁 Записать снова» (вызывает onRebook(client, lastService)); галерея фото до/после (все фото из визитов клиента, безопасные blob-URL с revoke).
- App: проброс onRebook → открыть форму записи с автозаполнением (clientName, contact, последняя услуга/цена).

### G3: Карточка записи — лояльность + отзыв
**Files:** `src/components/AppointmentCard.jsx`, `src/lib`, `src/styles.css`.
- Показать бейдж лояльности клиента (по визитам этого клиента — listByClient).
- Кнопка «🌟 Запросить отзыв» → WhatsApp с `reviewRequestText` (reviewLink из meta); скрыть если нет телефона.

### G4: Главная — сервис-блоки и аналитика
**Files:** `src/components/HomeView.jsx`, `src/styles.css`.
- Блок «🔔 Напомнить завтра»: записи на завтра, по каждой кнопка WhatsApp (reminderText). Пусто → скрыть/«на завтра записей нет».
- Блок «🎂 Дни рождения (30 дней)»: upcomingBirthdays, по каждому кнопка «Поздравить» (WhatsApp birthdayText). Скрыть если пусто.
- Блок «📊 Откуда приходят»: sourceStats — список источник→кол-во.

### G5: Раздел «Продвижение»
**Files:** переименовать/расширить `src/components/BroadcastView.jsx` → раздел с под-вкладками ИЛИ новый `PromoView.jsx`, обновить BottomNav (ярлык «Продвижение», иконка 📣), App. `src/styles.css`.
- Под-секции (простые табы внутри раздела): 
  1. «Рассылка» — существующая логика.
  2. «Свободные окошки» — выбрать день (сегодня/завтра), показать пост `buildFreeSlotsPost`, кнопка «Скопировать» + «Поделиться» (navigator.share при наличии).
  3. «Акции» — `promoTemplates`: выбрать шаблон → перенести в рассылку/скопировать.
  4. «Прайс-лист» — сгенерировать картинку прайса (canvas → PNG, услуги+цены, businessName), кнопка «Скачать/Поделиться».
- BottomNav: вкладка broadcast → «Продвижение».

### G6: Полировка + проверка + передеплой
- Дизайн-полировка новых блоков (единый стиль, тач-зоны, без внешних шрифтов).
- `npx vitest run` + `npm run build`; визуальная проверка; merge в main; пересборка GH_PAGES + публикация в gh-pages; проверка живого URL.
