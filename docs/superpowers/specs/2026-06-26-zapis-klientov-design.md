# Дизайн: приложение «Записи клиентов» (макияж и причёски)

Дата: 2026-06-26

## Цель

Удобное приложение для телефона (iPhone), в котором мастер по макияжу и
причёскам ведёт записи клиентов. Приложение должно работать прямо на её
телефоне и **не зависеть от компьютера владельца** после первичной публикации.

## Кто пользуется

Только мастер (один пользователь). Клиенты сами не записываются — они пишут ей
в директ/телефон, а она заносит запись в приложение.

## Архитектура

- **Тип:** PWA (Progressive Web App) — веб-приложение, добавляемое на экран
  «Домой» айфона. Открывается во весь экран как обычное приложение, работает
  офлайн.
- **Хостинг:** один раз публикуется на бесплатный статический хостинг (Netlify).
  После публикации не зависит от компьютера владельца.
- **Хранение данных:** локально в телефоне, в браузерной базе IndexedDB
  (записи, клиенты, услуги, фото как blob). Сервера и аккаунтов нет.
- **Язык интерфейса:** русский.
- **Дизайн:** крупные элементы под палец, мобильный-first, чистый и простой.

## Данные

### Запись (appointment)
- имя клиента
- телефон / ссылка на соцсеть
- дата и время начала
- услуга (из списка) + цена
- заметка (пожелания, аллергии и т.п.)
- фото результата (одно или несколько)
- ссылка на клиента (для истории визитов)

### Клиент (client)
- имя
- телефон / контакт
- история визитов (связанные записи)

### Услуга (service)
- название (макияж, причёска и т.п.)
- цена по умолчанию

## Экраны

1. **Сегодня / Ближайшие (главный)** — список записей по дням, ближайшие
   сверху. Видно имя, время, услугу, цену.
2. **Календарь** — месяц с отметками в дни с записями; тап по дню → записи дня.
3. **Новая запись** — форма со всеми полями выше + кнопка «➕ В Календарь
   iPhone».
4. **Карточка записи** — просмотр / редактирование / удаление; быстрые кнопки
   «позвонить», «написать в WhatsApp/Telegram».
5. **Клиенты** — список клиентов, по каждому история визитов (прошлые услуги и
   заметки при повторном визите).
6. **Настройки** — управление списком услуг и ценами, экспорт/импорт резервной
   копии.

## Напоминания

Приложение не показывает push само (ограничение веб-приложений на iOS).
Вместо этого кнопка «➕ В Календарь iPhone» создаёт событие в стандартном
Календаре (через файл .ics / ссылку), а Календарь сам надёжно напоминает за
заданное время. Серверов не требуется.

Push-уведомления напрямую от приложения — возможное расширение в будущем
(требует бесплатного облачного бэкенда, не компьютера владельца). В текущий
объём не входит.

## Резервная копия

Данные хранятся только в телефоне, поэтому есть риск потери при сбросе/потере
телефона. Решение:
- **«Сохранить копию»** — выгрузка всех записей + фото одним файлом в
  «Файлы»/iCloud.
- **«Загрузить копию»** — восстановление из файла.
Рекомендация пользователю — делать копию раз в неделю.

## Технологии

- React + Vite
- IndexedDB для хранения (через лёгкую обёртку, напр. idb)
- Конфигурация PWA (манифест + service worker) для установки на экран «Домой» и
  офлайн-работы
- Деплой на Netlify (бесплатно)

## Проверка

- Прогон основных сценариев в браузере: создать / найти / изменить / удалить
  запись; добавление в календарь; экспорт и импорт резервной копии; работа
  офлайн.
- Финальное тестирование на реальном iPhone мастера перед боевым
  использованием.

## Вне объёма (YAGNI)

- Самостоятельная запись клиентов онлайн.
- Push-уведомления напрямую от приложения.
- Многопользовательский доступ / облачная синхронизация между устройствами.
- Онлайн-оплата.
